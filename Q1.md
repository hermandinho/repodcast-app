> **#5 audiogram / waveform video**. Zero-budget path — one small VPS is

## Q1 goals (in priority order)

1. **Ship clip generation (#1) end-to-end** — 3–5 vertical 9:16 clips per
   episode, captions burned in, delivered from R2, previewable + trimmable
   in the app, publishable through the existing Buffer path (or manual
   download for platforms Buffer doesn't cover).
2. **Ship AI episode artwork (#4)** — square/horizontal/vertical variants
   per episode, wired into the existing outputs list (BLOG hero,
   NEWSLETTER hero, YouTube thumbnail candidates).
3. **Ship audiograms (#5)** — 30–60s waveform video for every social
   output as a "publish with audio" option, using the same render worker.

Non-goals for Q1: Remotion, GPU rendering, Cloudflare Stream, native
publishing to Reels/Shorts/TikTok video (Q3), post-publish analytics
(Q2).

## Infrastructure setup (do this first — week 1)

**Principle.** The box is bare-metal (already provisioned by hand — you
buy the VPS from Hetzner or use an existing OCI ARM instance). Everything
that happens on it after `ssh root@ip` is a checked-in shell script.
Never edit files on the box by hand; if something is missing, add it to
`bootstrap.sh` and re-run.

### 1. Repo layout

New top-level `worker/` folder in this repo (single-repo, no separate
worker repo to sync):

```
worker/
  Dockerfile                     # node:22-alpine + ffmpeg + fonts
  docker-compose.yml             # single service, restart unless-stopped
  package.json                   # Fastify + ytdl-core + AWS SDK
  src/
    server.ts                    # Fastify — POST /render/clip, /render/audiogram
    jobs/clip.ts
    jobs/audiogram.ts
    lib/ffmpeg.ts
  scripts/
    bootstrap.sh                 # first-run host setup: docker, ufw, fail2ban, deploy user
    deploy.sh                    # runs on the host: docker compose pull && up -d
    push-env.sh                  # local: scp .env → /home/deploy/worker/.env
    remote.sh                    # local wrapper: ssh deploy@$HOST -- "$@"
  Makefile                       # bootstrap, deploy, logs, ssh, health
  .github/workflows/
    worker-image.yml             # build + push to ghcr.io on push to main
    worker-deploy.yml            # SSH to host → run scripts/deploy.sh
```

### 2. Buy the box (5 minutes, once)

- **Recommended: Hetzner CPX11** (€4.35/mo, Ashburn) — 2 vCPU AMD,
  2 GB RAM, 40 GB NVMe. Add an SSH key at signup so the box is reachable
  without a password.
- **Zero-cost: Oracle Cloud Always Free ARM** (`VM.Standard.A1.Flex`,
  4 vCPU + 24 GB RAM). Longer signup, same end result.

Write the box's IP into `WORKER_HOST` (GitHub secret + local `.envrc`).
That's the whole "provisioning" step.

### 3. First-run host setup — `scripts/bootstrap.sh`

Idempotent shell script. Run once from your laptop when you get the
box, and again any time you change what needs to be installed. Nothing
in it depends on running in a fresh state — everything is `apt install`
(no-op if present), `usermod -a` (no-op if already in group), etc.

```bash
#!/usr/bin/env bash
# Usage: ./scripts/bootstrap.sh root@<ip>
# Idempotent — safe to re-run any time.

set -euo pipefail
TARGET="${1:?usage: bootstrap.sh root@<ip>}"

ssh -o StrictHostKeyChecking=accept-new "$TARGET" 'bash -s' <<'REMOTE'
set -euo pipefail

# --- packages ---
export DEBIAN_FRONTEND=noninteractive
apt-get update -qq
apt-get install -y -qq \
  ca-certificates curl gnupg ufw fail2ban unattended-upgrades

# Docker (official repo, not distro's outdated one)
if ! command -v docker >/dev/null; then
  install -m 0755 -d /etc/apt/keyrings
  curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
    | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
  chmod a+r /etc/apt/keyrings/docker.gpg
  echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release; echo $VERSION_CODENAME) stable" \
    > /etc/apt/sources.list.d/docker.list
  apt-get update -qq
  apt-get install -y -qq docker-ce docker-ce-cli containerd.io docker-compose-plugin
fi

# --- deploy user ---
if ! id deploy &>/dev/null; then
  useradd -m -s /bin/bash -G docker,sudo deploy
  echo 'deploy ALL=(ALL) NOPASSWD:ALL' > /etc/sudoers.d/deploy
  chmod 0440 /etc/sudoers.d/deploy
fi
mkdir -p /home/deploy/.ssh /home/deploy/worker
# Copy root's authorized_keys → deploy on first run only
[ -s /home/deploy/.ssh/authorized_keys ] || \
  cp /root/.ssh/authorized_keys /home/deploy/.ssh/authorized_keys
chown -R deploy:deploy /home/deploy/.ssh /home/deploy/worker
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys

# --- sshd hardening ---
cat > /etc/ssh/sshd_config.d/10-repodcast.conf <<EOF
PasswordAuthentication no
PermitRootLogin prohibit-password
KbdInteractiveAuthentication no
EOF
systemctl reload ssh

# --- fail2ban ---
cat > /etc/fail2ban/jail.d/sshd.local <<EOF
[sshd]
enabled = true
maxretry = 3
bantime = 1h
EOF
systemctl enable --now fail2ban

# --- ufw ---
ufw --force reset >/dev/null
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 443/tcp
ufw --force enable

# --- unattended security upgrades ---
dpkg-reconfigure -f noninteractive unattended-upgrades
systemctl enable --now unattended-upgrades

# --- docker daemon: json log rotation ---
mkdir -p /etc/docker
cat > /etc/docker/daemon.json <<EOF
{
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" }
}
EOF
systemctl restart docker
systemctl enable docker

echo "bootstrap: OK"
REMOTE
```

**One command from your laptop:**

```bash
./worker/scripts/bootstrap.sh root@<ip>
```

Adds ~2 minutes worth of package install to a fresh box; ~5 seconds on
a re-run. If the box ever gets weird, re-run it — it's a checked-in
declarative description of the host, just written in bash.

### 4. Compose file — the only long-lived state on the host

`worker/docker-compose.yml` is `scp`'d to `/home/deploy/worker/` once,
then updated in place by the deploy workflow. The `.env` file next to
it holds the runtime secrets, sourced by compose automatically.

```yaml
services:
  render:
    image: ghcr.io/${GHCR_OWNER}/repodcast-worker:${IMAGE_TAG:-main}
    restart: unless-stopped
    ports: ["443:8080"] # caddy sidecar later; for now, plain http on 443 through a reverse proxy is out of scope — put behind Cloudflare Tunnel instead (see §7)
    env_file: .env
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:8080/healthz"]
      interval: 30s
      timeout: 5s
      retries: 3
    logging:
      driver: json-file
      options: { max-size: 10m, max-file: "3" }
```

### 5. Deploy — GitHub Actions + `deploy.sh`

**Build (`.github/workflows/worker-image.yml`):** on push to `main`,
build `worker/Dockerfile`, push to `ghcr.io/{owner}/repodcast-worker`
tagged `main` and `sha-{sha}`. Multi-arch (amd64 + arm64) so switching
to Oracle ARM is one env change.

**Deploy (`.github/workflows/worker-deploy.yml`):** after the image
job succeeds, SSH to the box using `webfactory/ssh-agent` and run
`scripts/deploy.sh <tag>`.

`worker/scripts/deploy.sh` — lives on the host at
`/home/deploy/worker/deploy.sh`, kept in sync by the workflow:

```bash
#!/usr/bin/env bash
set -euo pipefail

IMAGE_TAG="${1:-main}"
cd /home/deploy/worker

echo "$GHCR_TOKEN" | docker login ghcr.io -u "$GHCR_USER" --password-stdin
export IMAGE_TAG
docker compose pull
docker compose up -d --remove-orphans
docker image prune -f
```

The workflow copies the latest `docker-compose.yml` and `deploy.sh`
into place before running:

```yaml
- run: scp -o StrictHostKeyChecking=no worker/docker-compose.yml worker/scripts/deploy.sh deploy@${{ secrets.WORKER_HOST }}:/home/deploy/worker/
- run: ssh deploy@${{ secrets.WORKER_HOST }} 'bash /home/deploy/worker/deploy.sh sha-${{ github.sha }}'
```

Secrets: `WORKER_HOST`, `WORKER_SSH_KEY`, `GHCR_TOKEN`, `GHCR_USER` —
all in GitHub Secrets.

### 6. Local ops — `make` targets

`worker/Makefile` — the only commands a human ever runs:

```makefile
HOST ?= $(WORKER_HOST)

bootstrap: ; ./scripts/bootstrap.sh root@$(HOST)
push-env:  ; ./scripts/push-env.sh deploy@$(HOST) .env.production

restart:   ; ssh deploy@$(HOST) 'docker compose -f /home/deploy/worker/docker-compose.yml restart'
```

Green-field flow from an empty Hetzner account:

```bash
# 1. Sign up + buy CPX11, save the IP
export WORKER_HOST=203.0.113.42

# 2. One-time host setup (~2 min)
make -C worker bootstrap

# 3. Push .env with R2 keys, Deepgram key, worker shared secret
make -C worker push-env

# 4. Trigger the deploy workflow (or just push to main)
make -C worker deploy

# 5. Verify
make -C worker health          # → 200 OK
```

**Total human-touched commands after the initial signup: four.** No
Terraform, no cloud-init, no Ansible. Everything is `bash` and
`docker compose`.

### 7. Reaching the box from the internet

CPX11 gets a public IPv4. Two options for TLS on `:443` — no reverse
proxy on the host, keep it Docker-only:

- **Recommended: Cloudflare Tunnel.** Add a `cloudflared` service to
  the compose file. Free, no port opening, no cert management, and DNS
  is handled inside Cloudflare (`render.repodcast.com`). Tunnel token
  lives in `.env`. Zero extra host config.
- **Alternative: Caddy sidecar.** Second service in compose, auto-ACME
  on Let's Encrypt. Requires opening 80 + 443 on the host firewall
  (already handled by bootstrap).

Pick one in week 1. Cloudflare Tunnel is the simpler answer for a
single-box setup.

### 8. Observability

- **Sentry.** `@sentry/node` in the worker, new project
  `repodcast-worker`, DSN in `.env`.
- **Logs.** `docker compose logs -f` when investigating. Ship-to-remote
  is deferred — Sentry catches errors, and the volume doesn't justify
  Grafana Cloud until #1 is generating hundreds of clips/day.
- **Healthcheck.** `GET /healthz` returns 200 + `{ lastRenderAt,
queueDepth }`. UptimeRobot free tier pings every 5 min.

### 9. Render worker service (`worker/`)

Contents of the container (built by GitHub Actions, deployed via
compose):

- `Dockerfile` — `node:22-alpine` + `apk add ffmpeg fontconfig
ttf-dejavu` (ffmpeg binary + system font for burnt-in captions).
- `src/server.ts` — Fastify HTTP endpoint (`POST /render/clip`,
  `POST /render/audiogram`, `POST /render/artwork-composite` if
  needed). Bearer-token auth using a shared secret stored in
  `WORKER_SHARED_SECRET`.
- `src/jobs/clip.ts` — download source, run ffmpeg, upload to R2.
- `src/jobs/audiogram.ts` — same, ffmpeg `showwaves` pipeline.
- `docker-compose.yml` — single service, `restart: unless-stopped`,
  healthcheck, log rotation (`max-size=10m,max-file=3`).
- **Concurrency:** in-process queue capped at 2 jobs (CPX11 CPU-bound).
  Longer queue lives in Inngest.
- R2 access via `@aws-sdk/client-s3` (R2 is S3-compatible), token
  injected as env — never baked into the image.

**Why a separate service, not Vercel:** ffmpeg jobs run 30s–2min and
Vercel functions cap at 5 min with cold-start Chromium overhead.
Splitting the worker also isolates the ffmpeg attack surface from the
app.

### 10. Inngest ↔ worker bridge

Extend `inngest/functions/` with:

- `generate-clips.ts` — triggered by `episode/clips.requested`. Calls
  the VPS worker, polls for completion via return payload, writes rows
  to `VideoClip`, emits `output/clip.rendered`.
- `generate-audiogram.ts` — triggered per social output when the user
  chooses "publish with audio."
- `generate-artwork.ts` — Workers AI is called directly from Inngest
  (no VPS needed), then the resulting bytes are uploaded to R2.

Reuse the existing `step.run` + retry semantics; don't invent a new
retry layer.

### 11. Cloudflare Workers AI

- Enable Workers AI on the existing Cloudflare account (already used
  for R2). Free tier is 10k neurons/day, plenty for artwork volume.
- Create a Workers AI API token: `Account → Workers AI → Read` scope.
- Model choice: `@cf/black-forest-labs/flux-1-schnell` (fast, decent
  quality). Fallback: `@cf/stabilityai/stable-diffusion-xl-base-1.0`
  when Flux is rate-limited.
- Called via REST (`https://api.cloudflare.com/client/v4/accounts/{acct}/ai/run/{model}`)
  from an Inngest step — no SDK needed.

### 12. Secrets and env vars

Two layers. Nothing is edited on the VPS by hand — the `.env` file on
the box is `scp`'d from your laptop (or from GitHub secrets in CI), never
composed manually with `nano`.

**Next.js app (`.env.example`, populated in Vercel):**

```

```

**Worker `.env` on the box (compose `env_file: .env`, kept at

```

R2_ACCOUNT_ID / R2_ACCESS_KEY_ID / R2_SECRET_ACCESS_KEY / R2_BUCKET

DEEPGRAM_API_KEY                # for transcript refetch when re-rendering
```

keep it (1Password, `gopass`, etc.), then `make -C worker push-env`.

the change. Total time: seconds.

### 13. YouTube source video ingestion

`@distube/ytdl-core` — the maintained fork. Vanilla `ytdl-core` breaks
weekly. Add to the worker's `package.json`, not the Next app (keeps
Vercel bundle lean).

Rate-limit handling: YouTube throttles unauthenticated requests
aggressively. First-pass without cookies; if 429s become common, mount
a `cookies.txt` on the VPS and pass to ytdl-core via env.

## Feature #1 — Short-form video clip generation

### Data model (Prisma migration — week 2)

```prisma
model VideoClip {
  id             String   @id @default(cuid())
  episodeId      String
  episode        Episode  @relation(fields: [episodeId], references: [id], onDelete: Cascade)
  startMs        Int
  endMs          Int
  score          Float
  hookLine       String   // Claude's pitch for why this moment
  sourceVideoUrl String?  // null → audiogram-style render
  renderedUrl    String?  // R2 signed URL after worker finishes
  posterUrl      String?
  captionsUrl    String?  // SRT in R2 (for re-render + edit)
  renderStatus   ClipRenderStatus @default(PENDING)
  renderError    String?
  agencyId       String   // denormalized for tenant filters
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  @@index([episodeId])
  @@index([agencyId])
}

enum ClipRenderStatus {
  PENDING
  RENDERING
  READY
  FAILED
}
```

Extend `Episode`:

```prisma
model Episode {
  // ... existing fields
  sourceVideoUrl  String?  // populated during import for YOUTUBE + UPLOAD-with-video
}
```

Do **not** touch the `Platform` enum yet — Q1 exposes clips as
downloadable assets; native publishing to REELS/SHORTS/TIKTOK_VIDEO
comes in Q3 (#10).

### Highlight selection (weeks 2–3)

New file: `server/ai/highlight-selection.ts`

- Input: Deepgram `words[]` with per-word timestamps + episode title.
- Prompt: reuse the shape in `server/ai/key-moments.ts` — extend for
  scoring on (hook strength, standalone comprehension, emotional beat,
  length window 25–75s).
- Output shape (enforced via Anthropic tool-use):
  ```ts
  {
    clips: Array<{ startMs; endMs; score; hookLine }>;
  }
  ```
- Cap at top 5. Validate spans don't overlap. Reject spans <15s or >90s.

### Render pipeline (weeks 3–6)

`worker/src/jobs/clip.ts` steps:

1. **Fetch source.** If `sourceVideoUrl` is a YouTube URL → ytdl-core to
   MP4 at 720p (adaptive: `videoformat=mp4, quality=highestvideo`). If
   R2 URL → stream direct. Cache the download for the whole batch of
   clips per episode (don't redownload for each clip).
2. **Trim.** `ffmpeg -ss {start} -to {end} -c copy` for stream-copy
   speed. Fall back to re-encode if keyframe alignment is off.
3. **Crop to 9:16.** `crop=ih*9/16:ih` + `scale=1080:1920`. For
   audio-only sources, use `showwaves` over a blurred artwork
   background (same code path as #5).
4. **Burn captions.** Deepgram → SRT (already available), then
   `subtitles=captions.srt:force_style='Fontname=DejaVu Sans,FontSize=28,PrimaryColour=&Hffffff&,Outline=2'`.
5. **Poster frame.** `ffmpeg -ss 0 -vframes 1` from the rendered MP4.
6. **Upload.** MP4 + poster to R2 under `clips/{agencyId}/{episodeId}/{clipId}.mp4`.

Target render time: **≤60s per clip on CPX11**. If slower, drop input
resolution to 540p before crop.

### UI (weeks 5–7)

New route: `app/(dashboard)/episodes/[id]/clips/page.tsx`

- Renders all `VideoClip` rows for the episode.
- Per-clip card: poster + `<video>` element, hook line, "Regenerate,"
  "Trim," "Download," "Copy shareable link" (24h signed R2 URL).
- **Trim editor** (v1 scope): number-input in/out fields, "Re-render"
  button. Skip `wavesurfer.js` for v1 — timeline scrubber is v2.
- Empty state: "Generate clips" button that fires
  `episode/clips.requested`.

### Test coverage

- Highlight selection unit test — feed a known transcript, assert
  clips are non-overlapping and within length bounds.
- Worker integration test — mock ytdl fetch, verify ffmpeg command
  string is well-formed. Skip actual encode in CI; run once nightly
  against a 60s fixture on the VPS.
- Tenant-isolation test — extend `tests/tenant-isolation` with
  `VideoClip` (must never leak across agencies).

## Feature #4 — AI episode artwork (parallel — weeks 3–4)

Standalone from #1 — different code path, different infra. Slot in
during the ffmpeg pipeline build.

### Data model

Extend `Episode`:

```prisma
model Episode {
  // ... existing fields
  heroImageUrl       String?  // 16:9 — YouTube thumbnail, blog hero
  squareCoverUrl     String?  // 1:1 — Apple Podcasts, IG feed
  verticalCoverUrl   String?  // 9:16 — Reels/TikTok cover
}
```

New table (optional — for user-picks-from-candidates flow):

```prisma
model EpisodeImageVariant {
  id           String   @id @default(cuid())
  episodeId    String
  episode      Episode  @relation(fields: [episodeId], references: [id], onDelete: Cascade)
  aspectRatio  String   // "16:9" | "1:1" | "9:16"
  imageUrl     String
  chosen       Boolean  @default(false)
  prompt       String   // for regenerate-with-tweaks
  createdAt    DateTime @default(now())

  @@index([episodeId])
}
```

### Pipeline

`inngest/functions/generate-artwork.ts`:

1. Given an episode, load transcript + key moments + show voice
   description.
2. **Prompt Claude** for a visual concept: `{ subject, mood, palette,
textOverlay? }`. Two-stage keeps the image-model prompt tight.
3. **Call Workers AI** for each aspect ratio (3 requests → 3 images).
4. Upload PNG bytes to R2 under `artwork/{agencyId}/{episodeId}/`.
5. Update `Episode` with the URLs.

### UI

- Show artwork on episode detail page + on BLOG / NEWSLETTER output
  previews (they currently ship without a hero).
- "Regenerate artwork" action (small button next to the images).
- No candidate picker in v1 — pick one per aspect, ship it. Add
  candidate grid in Q4 if churn signals demand it.

### Style anchoring

- Pass the show's existing `artworkUrl` in the Claude prompt as visual
  reference language ("in the style of the show's cover, which is
  {desc}"). No image-to-image on Workers AI in v1.
- Once you have >100 images per show, revisit LoRA/style transfer.

## Feature #5 — Audiogram / waveform video (week 8)

Piggybacks on the render worker from #1. Ships the week after clip
generation clears the gate.

### Endpoint

`POST /render/audiogram` on the worker:

- Input: audio URL, start/end ms, caption SRT slice, show
  `artworkUrl`, aspect ratio (`1:1` or `9:16`).
- ffmpeg filter graph:
  ```
  -loop 1 -i background.jpg  (blurred show artwork)
  -i audio.mp3
  -filter_complex "[1:a]showwaves=s=1080x300:mode=cline:colors=white[wave];
                   [0:v][wave]overlay=0:main_h-h[vid];
                   [vid]subtitles=captions.srt:force_style='...'"
  -shortest -c:a aac
  ```
- Extract dominant color for the background with `node-vibrant` at
  request time (cache per-show).

### UI

- On any social output drawer, add "Publish with audio" toggle.
- When enabled, generate audiogram at approve time.
- Attach the resulting MP4 to the Buffer post (or expose download for
  platforms Buffer doesn't cover).

### Data model

Reuse `VideoClip` with `renderStatus` — audiograms are just clips
with `sourceVideoUrl=null`. No new table.

## Risks and open questions

- **YouTube blocks ytdl-core.** Mitigation: fall back to yt-dlp binary
  on the VPS (drop-in via subprocess). Longer-term, offer users a
  manual video upload option for their podcast.
- **CPX11 render times exceed 60s.** Mitigation: (a) lower input res,
  (b) two-machine pool at €8.70/mo, (c) Modal.com on-demand for
  peak-hour spillover.
- **Workers AI free tier exhaustion.** 10k neurons/day = ~500 Flux
  requests (varies). At 3 aspects per episode = ~160 episodes/day
  ceiling. Monitor and switch peak-hour episodes to Hugging Face
  Inference API (also free-tier) when approaching cap.
- **Caption font licensing.** DejaVu Sans is Bitstream Vera derivative
  — permissive. If we ever want Inter (already in app), bundle the
  OFL license file with the worker image.
- **Signed R2 URLs vs. public bucket.** Signed URLs (24h expiry) for
  agency downloads. Public bucket path for Buffer attachments (Buffer
  fetches once and re-hosts). Don't mix.
- **VPS uptime.** No HA in Q1 — single node. Downtime = "clip
  generation temporarily unavailable" in-app banner. Acceptable at
  this stage.
